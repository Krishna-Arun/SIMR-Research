1️⃣ Context: What This Benchmark Tests

Dataset: 4 MIMIC-IV acute kidney injury cases with KDIGO 2012 classification

Task: Predict the next major procedural intervention given patient context WITHOUT lab values in the stem.

Why It Matters: Tests whether AI agents ground clinical reasoning in evidence (KDIGO guidelines + PubMed citations) and make patient-specific lab requests.

Conditions:

  • WITH_LAB_TOOLS: Agent can query MCP tools for specific labs
  • NO_TOOLS: Agent must reason from clinical context alone

2️⃣ Scoring: How It's Graded

Lab Score (0.70 max)
R_core (0.35) + R_wrong (0.20) + R_under (0.10) + R_over (0.20) + R_justification (0.15)
Answer Score (1.00 max)
c1 (correct procedure) + c2 (lab use) + c3 (causal chain) + c4 (patient accuracy)

Total = lab_score + answer_score (max 1.70)

6️⃣ How Questions Are Created (Evaluator-Optimizer Loop)

AKI benchmarks use an iterative generation loop (up to 5 rounds) where questions are automatically generated and strictly evaluated against 4 criteria. Only questions that PASS all criteria are approved for the benchmark.

The 4-Criterion Evaluation Process:

Criterion A: Clinically Sound Context — Question presents realistic clinical scenario without lab values. No contrived situations.

Criterion B: Requires Labs — Question must be UNANSWERABLE from clinical context alone. Labs are the critical missing piece.

Criterion C: Correct Lab Classification — L_core labs must be essential for the decision and grounded in published guidelines (KDIGO 2012 + PubMed citations). L_optional and L_irrelevant must have clear justifications.

Criterion D: Patient Accuracy — Demographics, diagnoses, and clinical facts must match the actual patient from MIMIC-IV. No invented facts.

Example: Patient 10039708 Generation Loop

Round 1: Generated question → REJECTED (Criterion C failed. Lab classification did not cite KDIGO Section 5.3 RRT indications.)

Round 2: Regenerated with explicit KDIGO Section 5.3 citations (Cr ≥4.0, K+ ≥6.5, HCO3- <10) + PubMed PMIDs → PASSED all 4 criteria

Rubric Building: Once a question passes all 4 criteria, a detailed answer rubric is built (c1: correct procedure, c2–c4: required labs, causal reasoning, patient context).

7️⃣ How Questions Are Graded (Two-Stage Process)

Stage 1: Automatic Lab Scoring

lab_score = min(0.70, 0.35·R_core + 0.20·R_wrong + 0.10·R_under + 0.20·R_over + 0.15·R_justification)

R_core (0.35): Fraction of L_core labs correctly requested + interpreted with KDIGO thresholds

R_wrong (0.20): Penalty for L_irrelevant labs cited as primary drivers

R_under (0.10): Exponential penalty for missing L_core labs

R_over (0.20): Exponential penalty for excessive lab requests

R_justification (0.15): 0 (absent) | 0.5 (generic) | 1.0 (patient-specific, tied to THIS patient's condition)

Stage 2: Supervisor Grading (Three-Agent Team)

Reward Agent: Scores c1–c4 on 0–1 scale per the rubric

Critic Agent: Explains WHY each score was assigned; identifies gaps and strengths

Optimizer Agent: Placeholder for future RL optimization; suggests how answer could have scored higher

Final Score: Total = lab_score (0–0.70) + answer_score (0–1.00) = max 1.70

1️⃣ Context: What This Benchmark Tests

Dataset: 100 EHRSHOT cardiac cases with direction-reversal windows

Task: Predict next serial Troponin I value and direction when visible trend reverses.

Why Hard: Last-value heuristic AND trend-extrapolation both fail by construction. Requires cross-lab signal integration (BNP, creatinine, lactate, vitals).

Four Reversal Types: Rising→Falling (35), Falling→Rising (35), Stable→Rising (15), Stable→Falling (15)

2️⃣ Scoring: How It's Graded

Direction Correct (0.40)
Binary: predicted direction == ground truth (threshold: >20% change)
Within 50% Error (0.35)
|predicted - actual| / actual ≤ 0.50
Within 20% Error (0.25)
|predicted - actual| / actual ≤ 0.20

Max score: 1.0 | Baseline (random direction): 33% accuracy

5️⃣ Key Findings

1. Direction Detection >> Value Precision: 82% direction accuracy (far above 33% random), but only 31% within 50% value error.

2. Upward Reversals Much Easier: Rising→Falling (91%) and Stable→Rising (93%) vs Stable→Falling (33%). Rising bias dominates.

3. Magnitude Anchoring: Falling→Rising cases: 100% direction accuracy BUT only 3% within 50% error. Agent anchors on last low value, misses future peak.

4. Recovery Signals Hard: Stable→Falling requires detecting subtle recovery signals (falling BNP, improving Cr) without visible momentum. Agent defaults to rising.

6️⃣ How Questions Are Created (Case Preparation)

Data Source: 100 cardiac cases selected from EHRSHOT database (ACS + HF patients with complete troponin trajectory data).

Direction-Reversal Window Design: For each patient, identify the visible troponin trend (rising, falling, or stable), then select a future timepoint where the trend reverses. This ensures both last-value AND trend-extrapolation heuristics fail by construction.

Data Leakage Prevention: The ground-truth troponin value at the target time is hidden from the agent during prediction. Cross-lab signals (BNP, creatinine, lactate, vitals) available up to the target time, but the troponin value itself is not revealed until scoring.

Case Distribution: Stratified by reversal type to ensure balanced difficulty assessment: Rising→Falling (35), Falling→Rising (35), Stable→Rising (15), Stable→Falling (15).

7️⃣ How Questions Are Graded

Scoring Formula (identical to v1):

Direction Correct = 0.40 (binary: predicted direction == ground truth, threshold >20% change)

Within 50% Error = 0.35 (|predicted - actual| / actual ≤ 0.50)

Within 20% Error = 0.25 (|predicted - actual| / actual ≤ 0.20)

Max Score = 1.0 | Baseline (random direction) = 33% accuracy

Stratified Analysis: Results reported separately by reversal type to identify where agents struggle most (e.g., Stable→Falling shows only 33% direction accuracy despite being clinically critical).

1️⃣ Context: What This Benchmark Tests

Dataset: 100 EHRSHOT cases (reorganized with explicit conditions)

Task: Predict next Troponin I value across different tool/context conditions.

Conditions:

  • Baseline: Labs only, no external tools (control)
  • With PubMed: Labs + MCP PubMed server access (test)

Purpose: Measure whether external literature access helps or confuses troponin prediction.

2️⃣ Scoring: How It's Graded

Identical to v1: Direction (0.40) + Within 50% (0.35) + Within 20% (0.25) = max 1.0

3️⃣ Sample Case: Case 1 (Rising → Falling)

Condition Comparison

Baseline: Predicted 3.5, Falling → Direction ✅ Score 0.40
With PubMed: Predicted 3.5, Falling → Direction ✅ Score 0.40

Both conditions yielded same answer. For this case, PubMed access made no difference.

4️⃣ Results: 20 Cases Across Conditions

Metric Baseline With PubMed
Mean Score 0.288 0.238
Direction Accuracy 20% (4/20) 15% (3/20)
Predicted "Rising" 12/20 (60%) 12/20 (60%)
Within 50% Error 45% (9/20) 40% (8/20)

5️⃣ Key Findings

1. PubMed Access Paradox: Baseline (0.288) OUTPERFORMS With PubMed (0.238). External tools sometimes confuse more than help.

2. External Literature Hurts: Adding PubMed access decreased both mean score (-5%) and direction accuracy (20% → 15%). Agent gets distracted by general literature instead of focusing on patient-specific cross-lab signals.

3. Rising Bias Persists: 60% predicted rising in both conditions. Not a prompt artifact — fundamental to agent heuristics.

4. Lesson: For troponin prediction, agent's intuition from lab patterns > ability to integrate external literature. Sometimes less is more.

6️⃣ How Conditions Are Designed

v2 vs v1: Dirchange v1 is a single-condition benchmark (agents have only lab data). Dirchange v2 adds experimental conditions to test whether external tools help or hurt prediction accuracy.

Baseline Condition: Labs only, no external tools. Agent has access to troponin trajectory, BNP, creatinine, lactate, vitals, medications — the full lab arsenal from EHRSHOT. This is the control group.

With PubMed Condition: Same lab data PLUS access to MCP PubMed server. Agent can search for literature on troponin kinetics, cardiac markers, HF pathophysiology, etc. The question: does external literature help agents make better cross-lab inferences, or does it introduce noise and distraction?

Key Insight: The paradoxical finding that PubMed access DECREASES performance (0.288 → 0.238) suggests that for troponin prediction, agent's pattern recognition from lab data > ability to integrate general literature. Agents may be pattern-matching against general troponin descriptions rather than focusing on THIS patient's specific multi-lab trajectory.

7️⃣ Scoring & Analysis

Scoring Formula: Identical to v1 (Direction 0.40 + Within 50% 0.35 + Within 20% 0.25 = max 1.0).

Within-Condition Analysis: For each case, we compare Baseline vs With PubMed predictions and scores. Same 20 cases, same evaluation criteria, different information access.

Aggregate Metrics: Mean score, direction accuracy, error distribution across both conditions. Charts show the stark performance gap and help diagnose whether PubMed access is genuinely distracting agents or if the decrement is noise (20 cases is a small sample).
