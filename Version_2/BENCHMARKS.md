# Version_2 — Medical-Reasoning Benchmarks (A / B / C)

A suite of three benchmarks that test whether an LLM agent can **reason causally** over a real
cardiac patient's data — not just recall facts. Each question is anchored to a real **MIMIC-IV
cardiovascular ICU/hospital admission** and is generated by an Evaluator–Optimizer pipeline with
agentic access to PubMed.

## Reasoning, defined (working definition)
Reasoning = building an **interpretable causal chain** from a query down to the specific pieces of
patient evidence, where the links also hold *back up* (evidence → mechanism → conclusion). A good
answer cites the patient's own findings as the evidence at each step, not generic textbook statements.

## Cohort (shared)
- **Population:** real MIMIC-IV admissions with a **primary cardiac diagnosis** (`cardiovascular_primary`
  filter — coronary disease, valve disorders, MI, heart failure, arrhythmias, etc.; peripheral-vascular
  and cerebrovascular excluded). Effectively **all heart patients**.
- **Excluded:** patients **age ≥ 89** (MIMIC's 91 sentinel for 90+).
- **Supplemental data available per patient** (served on-demand via an agent tool, plus a brief
  names/dates summary the generator sees): **all labs, microbiology, medications**. (Chief Complaint /
  HPI / Physical Exam are omitted — `mimic-iv-note` is not available.)
- **Anti-leakage:** question stems never name the labs, values, diagnosis, or answer — the agent must
  request/derive them. Enforced by the evaluator's automatic checks.

## Models
- **Question generator (both Evaluator + Optimizer roles):** `openai/gpt-oss-20b` — a *neutral* family,
  deliberately **excluded from the test set** so it doesn't bias the benchmark.
- **Models under test:** Qwen3-27B, Gemma-27B (each run **with and without** the PubMed tool).
- **External tool:** PubMed MCP server (for guideline citations during generation; and as a tool the
  tested agent may use).
- Headline metric across all three: the **Δ in score with vs. without PubMed**.

## Two phases
- **Phase 1 — Question generation (built):** produces `outputs/questions.jsonl` per benchmark (the
  question + the gold answer key). Status: aligned + running.
- **Phase 2 — Agent evaluation + scoring (not built yet):** the request tools, the with/without-PubMed
  agent runs, and the scoring rubrics below. Scorer model TBD (not Llama-70B).

## Benchmark A — Lab-requesting + causal diagnosis/intervention
**Goal:** Can the agent figure out *which* labs/microbiology it needs (with patient-specific
justification) and then answer the clinical question with a causal explanation?

**Agent input**
- A text **diagnosis or intervention question** about one patient (no labs/values/diagnosis leaked).
- Tool `request_all_labs_no_values` → a table of available lab/microbiology **types + dates** (values
  hidden; includes irrelevant/supplemental items so the choice is meaningful).
- Tool `request_a_lab(name, justification)` → returns all values for that lab across all dates
  (must give patient-context justification). Medications are available as supplemental context.

**Expected output**
- The answer to the clinical question (diagnosis or suggested intervention) + a stated **confidence**.

**Scoring** (each 0 / 0.5 / 1)
- *Lab request:* 0 = irrelevant/wrong; 0.5 = accurate but generic/textbook; 1 = patient-specific
  justification, uses multiple labs appropriately.
- *Answer:* 0 = wrong/irrelevant; 0.5 = generic causal links, labs not used; 1 = patient-specific
  causal links using the labs (and external evidence).

**Gold key (generated):** `type`, `gold_labs[]` (label, patient_value, abnormal_flag, guideline
citation {pmid, claim}), `gold_micro[]`, `reference_answer`, `causal_chain[]`, `pubmed_citations`.

## Benchmark B — Post-procedure lab-trajectory prediction
**Goal:** Can the agent predict, with causal justification, how each core lab moves **after a
procedure** (and at inflection points)?

**Agent input**
- The patient's pre-procedure state: all labs + microbiology (via tool) + **medications**, prior
  **invasions** (procedures already done), and the **name(s) of the procedure(s) performed** (time-zero).
- The set of core labs (measured repeatedly before & after) whose trajectories must be predicted.

**Expected output**
- For each target lab: a **direction ∈ {Rising, Falling, Stable}** + causal justification, and a
  **confidence** (probability distribution over the three directions).

**Scoring** (added to A's answer rubric)
- 1 = correct direction; 0.5 = predicted Stable when actually non-stable; 0 = opposite direction
  (or any direction when truth is Stable).
- Plus **ECE** (calibration of stated confidence vs. the activation-probability distribution).

**Ground truth (data-derived, not model-invented):** `true_direction` per lab where **Stable = ends
within the lab's reference range**; otherwise direction = sign of movement vs. baseline (e.g. creatinine
1.7→1.4 but still above range = *Falling*). Stored with baseline, post value, ref range, pre/post series.

## Benchmark C — Counterfactual intervention identification
**Goal:** Given two clinically plausible interventions and one observed post-procedure state, can the
agent identify **which intervention actually occurred** and justify it via each procedure's causal
physiologic effects?

**Agent input**
- For **both** Patient A and Patient B: procedure name, prior **invasions**, comorbidities, and
  pre-procedure labs (+ microbiology + medications via tool).
- **One** patient's **observed post-procedure labs** (unlabeled — belongs to exactly one of them).

**Expected output**
- Which patient/procedure produced the post-state (**A or B**) + causal justification + **confidence**.

**Scoring** (added to A's answer rubric)
- 0 / 1 for correctly identifying the patient; plus **ECE** (confidence vs. activation-probability over {A,B}).

**Pairing / "overlap" method:** the two patients are **propensity-score matched** so their baselines
overlap, making the task non-trivial. Covariates = standardized latest pre-procedure core labs
(median-imputed) + comorbidity flags + age → logistic-regression propensity for each **cross-system**
procedure pair (e.g. renal dialysis vs. a respiratory procedure — only cross-system, so the two have
distinct lab signatures). 1:1 nearest-neighbour match within a caliper, kept only if the overlap
passes SMD + post-match-AUC (~0.5) quality gates. Ground truth = which patient owns the shown post-state.
